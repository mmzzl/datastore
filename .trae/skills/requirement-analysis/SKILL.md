# Requirement Analysis

This skill provides a structured approach to analyzing and documenting software project requirements.

## Overview

Requirement analysis is a critical phase in software development that ensures projects meet stakeholder needs and business objectives. This skill guides you through the entire requirement analysis process, from initial stakeholder interviews to final requirement documentation.

## When to Use

**Invoke this skill when:**
- Starting a new software project
- Defining features for an existing project
- Clarifying project scope and boundaries
- Resolving conflicting requirements
- Documenting business needs and technical constraints
- Planning for project estimation and scheduling

## Key Components

### 1. Stakeholder Identification

- Identify all relevant stakeholders
- Understand their roles and responsibilities
- Determine their level of influence and interest
- Establish communication channels

### 2. Requirement Gathering

- Conduct interviews with stakeholders
- Facilitate focus groups and workshops
- Analyze existing documentation
- Review industry standards and best practices
- Collect user feedback and pain points

### 3. Requirement Classification

- **Functional Requirements:** What the system should do
- **Non-Functional Requirements:** How the system should perform
- **Business Requirements:** Business objectives and constraints
- **User Requirements:** End-user needs and expectations
- **Technical Requirements:** Technical constraints and dependencies

### 4. Requirement Analysis

- Validate requirements with stakeholders
- Identify and resolve conflicts
- Prioritize requirements based on business value
- Assess feasibility and technical risk
- Define acceptance criteria

### 5. Requirement Documentation

- Create use cases and user stories
- Develop functional specifications
- Document non-functional requirements
- Create data models and process flows
- Establish requirement traceability

### 6. Requirement Management

- Implement change control processes
- Track requirement status
- Monitor requirement coverage
- Facilitate requirement reviews
- Update requirements as project evolves

## Best Practices

### Requirement Elicitation

- Use open-ended questions to encourage detailed responses
- Active listening to understand underlying needs
- Document requirements immediately to avoid loss of information
- Validate understanding by paraphrasing stakeholder responses
- Involve cross-functional teams for comprehensive input

### Requirement Documentation

- Use clear and unambiguous language
- Avoid technical jargon when communicating with non-technical stakeholders
- Document assumptions and constraints
- Establish measurable acceptance criteria
- Maintain version control for requirement documents

### Requirement Validation

- Conduct regular requirement reviews
- Use prototypes to validate user requirements
- Test requirements against business objectives
- Verify technical feasibility
- Ensure requirements are testable

## Example Workflow

1. **Initiation:** Define project scope and identify stakeholders
2. **Elicitation:** Gather requirements through interviews and workshops
3. **Analysis:** Classify, prioritize, and validate requirements
4. **Documentation:** Create comprehensive requirement documents
5. **Validation:** Review and refine requirements with stakeholders
6. **Management:** Implement change control and track requirements

- **Documentation Tools:** Google Docs, Microsoft Word
- **Prototyping Tools:** Figma, Sketch, Adobe XD

## Common Challenges

### Ambiguous Requirements
- **Solution:** Ask clarifying questions, use examples, and create prototypes

### Changing Requirements
- **Solution:** Implement change control processes, prioritize changes, and communicate impact

### Stakeholder Conflicts
- **Solution:** Facilitate meetings, focus on common goals, and escalate when necessary

### Incomplete Requirements
- **Solution:** Conduct thorough elicitation, use checklists, and validate with multiple stakeholders

### Unrealistic Requirements
- **Solution:** Assess feasibility, provide alternatives, and manage expectations

## Conclusion

Effective requirement analysis is the foundation of successful software projects. By following a structured approach, you can ensure that projects meet stakeholder needs, stay within scope, and deliver business value. This skill provides the framework and best practices to conduct comprehensive requirement analysis for any software project.
